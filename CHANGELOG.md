# Changelog

All notable changes to this project will be documented in this file.

The format is based on [Keep a Changelog](https://keepachangelog.com/en/1.1.0/),
and this project adheres to [Semantic Versioning](https://semver.org/spec/v2.0.0.html).

## [1.0.32] - 2026-03-01

### Added

- Integrated version management system (`version_manager.py`) with SQLite-backed release tracking
- System menu (option 7) in the main menu exposing version history, new release, and edit notes
- Auto-regeneration of `CHANGELOG.md` via `version_manager.update_changelog()` on every release
- `README.md`, `requirements.txt`, `.gitignore`, and `CHANGELOG.md` project scaffolding

### Changed

- Main entry-point file renamed from `notifier.1.0.32.py` to `notifier.py`

## [1.0.31] - 2025-12-14

### Fixed

- Desktop notification guard now checks `callable(getattr(notification, "notify", None))` to prevent crash when `plyer` is installed but `notification.notify` is not callable

### Changed

- Background scheduler failure is now caught and logged as a warning instead of crashing the app on startup

## [1.0.20] - 2025-10-09

### Added

- Pushover integration with `send_pushover_message()` and `verify_pushover_config()`
- `notification_services_menu()` as a unified hub for all platform submenus
- `show_complete_env_example()` helper to display a ready-to-copy `.env` template
- Per-service status indicators (`✅` / `❌`) shown inline in the services menu

## [1.0.10] - 2025-08-22

### Added

- Discord webhook integration with `send_discord_message()` and `verify_discord_config()`
- Gmail (SMTP) integration with `send_email_message()` and `verify_email_config()`
- App-password instructions displayed in the Gmail setup menu

## [1.0.0] - 2025-06-01

### Added

- Initial release with Telegram notification support
- SQLite-backed notification store (`notifications.db`)
- Background scheduler thread checking for due notifications every 60 seconds
- Full CRUD: add, view, edit, and delete scheduled notifications
- `colorama` color-coded terminal UI
- `plyer` desktop notification support with graceful fallback when unavailable
- `.env` driven configuration via `python-dotenv`
